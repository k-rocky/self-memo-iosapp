# アプリ企画メモ：感性を掘り起こすAIジャーナル

## 1. 背景

現代の大人、特に30代前後の知的労働者は、仕事・家庭・生活基盤をある程度整えていても、「自分は何に心が動くのか」「何に熱量を持てるのか」が分からなくなることがある。

多くの人は、社会適応・安定・責任・成果を優先して生きる中で、自分の感性や衝動を後回しにしている。その結果、日常に大きな不満はないが、「このままでいいのか」「自分の人生に自分の色があるのか」という漠然とした欠落感を抱える。

既存のメモアプリや日記アプリは記録には向いているが、記録された断片から「自分が何に生きている感覚を得るのか」を発見する支援は弱い。また、自己啓発アプリは正論・習慣化・目標管理に寄りがちで、感性や妄想を広げる余白が少ない。

本アプリは、日常の小さな気づきや違和感を起点に、AIが少し飛躍した問いを返すことで、ユーザーの想像力・感性・価値観を掘り起こすことを目的とする。

## 2. コンセプト

一言で言うと、

「日常のメモを、人生の感性ログに変えるAIアプリ」

または、

「ふと思ったことから、自分が生きている感覚を見つけるアプリ」

## 3. 解決したいペイン

### 表層的なペイン

* 日記を書いても続かない
* 自分が何をしたいのか分からない
* 毎日がタスク消化になっている
* 感情や違和感をうまく言語化できない
* 自分の興味・価値観がぼんやりしている

### 深層的なペイン

* 安定しているのに満たされない
* 自分の人生が“作品”になっていない感覚がある
* 正しく生きてきたが、自分の色が分からない
* 何かに貢献したいが、何に燃えるのか分からない
* 創造性や想像力が眠っている感覚がある
* 他人の熱量や表現力を見て、羨ましさや焦りを感じる

## 4. ターゲットユーザー

### メインペルソナ

30代前後の知的労働者。
仕事も家庭もある程度安定しているが、自分の内側の熱量や創造性が鈍っている感覚を持っている人。

#### 属性

* 年齢：28〜40歳
* 職種：IT、コンサル、企画、研究開発、医療系専門職、エンジニア、PMなど
* 家庭：既婚・子持ちも含む
* 特徴：

  * 真面目
  * 適応力が高い
  * 思考力がある
  * それなりに成果を出してきた
  * でも最近、自分の人生に物足りなさを感じている

#### 代表的な内面

* 「別に不幸ではない。でも、このままでいいのかと思う」
* 「自分が何に本当に心動くのか分からない」
* 「昔はもっと熱量があった気がする」
* 「創造的に生きたいが、何をすればいいか分からない」
* 「自己啓発っぽい正論は苦手。でも何か変えたい」

## 5. MVP定義

### MVPの目的

ユーザーが日常の一言メモから、AIによる問いかけを受け取り、それに答えることで、自分の感性・価値観・熱量のパターンを少しずつ発見できることを検証する。

### MVPで実装する機能

#### 1. 一言メモ入力

ユーザーが、その時ふと思ったことを短く入力する。

例：

* 「結婚式で友達の熱量に嫉妬した」
* 「宇宙のことを考えるとワクワクする」
* 「今日の仕事は意味を感じなかった」
* 「子どもと遊んでいて妙に幸せだった」

#### 2. AIによる問い生成

入力内容をもとに、AIが少し飛躍した問いを1つ返す。

問いの方向性：

* 妄想を広げる
* 感情の奥を掘る
* 価値観を見つける
* SF・科学・未来・人生観に接続する
* 正論ではなく、思考が動く問いにする

例：

入力：
「結婚式で友達の熱量に嫉妬した」

AI問い：
「あなたが羨ましかったのは、彼らの才能ですか？ それとも、自分たちの人生を自分たちの色で表現している感じですか？」

入力：
「宇宙のことを考えるとワクワクする」

AI問い：
「もしあなたが一つだけ人類の未来に貢献できるとしたら、それは“知ること”と“作ること”のどちらに近いですか？」

#### 3. 回答入力

ユーザーがAIの問いに対して自由記述する。

#### 4. ログ保存

以下を保存する。

* 日付
* ユーザーの一言メモ
* AIが返した問い
* ユーザーの回答
* 任意タグ
* 感情スコア、または熱量スコア

#### 5. 週次サマリ

一定数のログが溜まったら、AIが傾向をまとめる。

例：

* あなたは「未来」「仲間」「知的挑戦」に反応しやすい
* 「意味が見えない作業」に強い虚無感を感じやすい
* 「自分の能力が何かを前に進める感覚」が重要そう
* 最近は「創造性」「表現」「普通であることへの不安」が頻出している

## 6. MVPではやらないこと

初期版では以下は実装しない。

* SNS機能
* 他ユーザーとの共有
* 高度なグラフ可視化
* ゲーミフィケーション
* 課金
* 複雑なプロフィール診断
* 長期的なパーソナライズAI
* Apple Watch連携
* 音声入力
* カレンダー連携

理由：
最初に検証すべきなのは、「一言メモ → AIの問い → 自己洞察」という体験が刺さるかどうかであり、周辺機能は後回しでよい。

## 7. 技術的な検討事項

### フロントエンド

* iOSネイティブアプリ
* SwiftUIを想定
* まずはシンプルな画面構成でよい

主要画面：

1. ホーム画面
2. メモ入力画面
3. AI問い表示画面
4. 回答入力画面
5. ログ一覧画面
6. 週次サマリ画面

### バックエンド

初期案は2パターン。

#### 案A：ローカル中心 + API直接呼び出し

* iOSアプリ内にログ保存
* LLM APIをアプリから呼び出す
* MVPとしては最短

懸念：
APIキー管理が難しいため、公開アプリには不向き。

#### 案B：軽量バックエンド経由

* iOSアプリ
* 軽量APIサーバー
* LLM API
* DB

構成例：

* iOS：SwiftUI
* Backend：FastAPI / Node.js / Firebase Functions
* DB：Firebase Firestore / Supabase / PostgreSQL
* LLM：OpenAI APIなど

MVPでも、将来公開を考えるなら案Bが無難。

### AI処理

主なAI処理は2つ。

#### 1. 問い生成

入力メモから、ユーザーの感情・価値観・妄想の余地を踏まえて問いを生成する。

重要なのは、カウンセリング風にしすぎないこと。
正論や慰めではなく、「思考が少し斜め上に飛ぶ問い」を返す。

#### 2. サマリ生成

一定期間のログをもとに、ユーザーの傾向をまとめる。

注意点：
心理診断のように断定しすぎない。
「あなたはこうです」ではなく、「こういう傾向が見えます」程度にする。

## 8. 非機能要件

### プライバシー

扱う内容はかなり個人的になるため、プライバシーは重要。

考慮点：

* ログはユーザー本人だけが閲覧できる
* 不要な外部共有はしない
* AI処理に送信される内容を明示する
* アカウント削除時にデータ削除できる
* 初期版ではローカル保存も選択肢

### セキュリティ

* APIキーをアプリに直埋めしない
* 通信はHTTPS
* 認証を入れる場合はFirebase AuthやSign in with Appleを検討
* 個人ログのアクセス制御

### UX

* 起動してすぐ書ける
* 入力ハードルを低くする
* 1回の利用時間は1〜3分程度
* 重い自己分析アプリにしすぎない
* 夜に開きたくなる雰囲気
* 余白、静けさ、少しSF感のあるUI

### レスポンス速度

* 問い生成は数秒以内が望ましい
* 遅い場合は「問いを生成中」の演出を入れる
* 体験として待ち時間が苦痛にならない設計にする

### コスト

* LLM APIコストが発生する
* MVPでは1ユーザーあたりの利用頻度を想定してコスト試算が必要
* サマリ生成は毎回ではなく週次や手動実行にする

## 9. 成功指標

### MVP検証指標

#### 定量指標

* 1週間で3回以上メモしたユーザー割合
* 初回利用後、翌日以降も戻ってくる割合
* AI問いに対する回答率
* 1回あたりの平均回答文字数
* 週次サマリを閲覧した割合
* 継続利用日数

#### 定性指標

* 「自分の感性が少し見えた」と感じるか
* AIの問いが浅くないか
* 日記より書きやすいか
* 正論臭くないか
* また使いたいと思うか
* 自分だけのログを育てている感覚があるか

### 個人開発としての成功指標

* まず自分が1週間使いたいと思える
* 自分のログが10件以上溜まる
* 週次サマリを読んで腹落ちする発見が1つ以上ある
* 家族や友人1〜3人に触ってもらえる
* 作っていて自分自身がワクワクする

## 13. 現時点の一番大事な仮説

このアプリの最大の仮説は、

「人は、自分のふとした感情や違和感に対して、少し飛躍した問いを返されると、自分の感性や価値観を発見しやすくなる」

ということ。

最初に検証すべきなのは、この体験が本当に気持ちいいかどうか。

機能を増やすより先に、
AIが返す問いの質を磨くべき。
